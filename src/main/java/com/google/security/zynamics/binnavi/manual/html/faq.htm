FAQ - Frequently Asked Questions

What is required to run BinNavi?

BinNavi should work on every platform that can execute Java 6 applications.

What platforms does BinNavi support?

BinNavi officially supports code analysis of binary code from the following platforms:

  • All 32-bit x86 architectures
  • All 32-bit PowerPC architectures
  • All 32-bit ARM architectures
  • All 32-bit MIPS architectures

What platforms can be debugged with BinNavi?

So far BinNavi was successfully used to debug the following target applications:

  • x86-32 applications on Windows 2000, Windows XP, Windows Vista, Windows 7
  • x86-32 applications on Linux (Ubuntu)
  • x86-32 applications on 32 bit FreeBSD
  • PPC-32 applications on Cisco 2600 series routers (GDB debug client)
  • PPC-32 applications on NetScreen 5XT VPN firewalls (GDB debug client)
  • PPC-32 applications on a PowerPC 603e using the bdi2000 JTAG debugger (GDB debug client)

Troubleshooting

The assembly code is not layouted properly. What can I do?

This issue indicates that BinNavi did not manage to load a monospaced font. By default the font Courier New is loaded. If Courier New can not be found on the system a default monospaced font is loaded. On some systems there is no default monospaced font and a monospaced font must be installed manually.

After the installation of the monospaced font you have to tell Java to use the monospaced font. This can be done in the font properties file of your Java Runtime Environment. For more information about font properties files please see http://java.sun.com/j2se/1.5.0/docs/guide/intl/fontconfig.html . The change to the font properties file is not necessary if you simply install the Courier New from Windows on your system.
